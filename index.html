#1 State Mgmt

State == Single Source of truth

						const { auth } = useContext(AuthContext);

This yields 70 Results in 70 different files

What is actually happening here?

Context Provider:

AuthContext provides auth data (e.g., user details, authentication status) at a high level in the component tree. Every component that needs auth subscribes to this context using useContext(AuthContext).

Effects

Tight Coupling:

Components are tightly coupled to AuthContext. Any changes to the context's structure or logic will ripple through all dependent components.

Performance Issues:

When auth changes (e.g., user logs in or out), all components using useContext(AuthContext) re-render, even if they only use a small part of the data.

Scalability Problems:

As the app grows, having multiple useContext calls littered in the code becomes hard to manage. Debugging becomes difficult because it's unclear which components depend on the context.

Code Duplication:

Every component must explicitly import and subscribe to AuthContext, leading to redundancy and boilerplate.
Why is stage management preferable?

Global State Management:

Redux (or other state mgmt tool) creates a centralized global store that can hold auth and other app-wide state. Components access the auth state using the useSelector hook, which subscribes only to the specific part of the state needed.

Effects:

Decoupled Architecture: Components no longer depend directly on a context provider. They interact with the global store instead. This makes components more reusable and easier to test. Optimized Re-Renders: Redux's useSelector is highly optimized to trigger re-renders only for components that use the specific slice of state that changed. No unnecessary re-renders for unrelated components.
Scalability: Redux's modular architecture (e.g., slices) makes it easier to scale large apps. Adding new features or refactoring is more manageable because state logic is centralized. Code Cleanliness: No need for useContext calls in every component. Simply use useSelector((state) => state.auth) to access the auth state from anywhere in the app.
